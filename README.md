# JAZZPROMPT2MIDI: THE MOST IMPORTANT

A minimalist, high-fidelity Web Audio Svelte application designed to parse deeply humanized structural music data (generated by LLMs or humans) into rich acoustic piano performances and perfectly quantized multi-track `.mid` file exports.

![Screenshot of the App](https://via.placeholder.com/800x450.png?text=JazzGPT+MIDI+App) <!-- Replace with actual screenshot path later -->

## 🎹 Overview
The goal of this application is to take a rigorously structured prompt text that defines musical metadata, complex jazz chords, and micro-timed floating-point melody durations, and instantly synthesize it.

It acts as both a **Text-to-Audio** player and a **Text-to-MIDI** parser, built entirely on pure client-side Vanilla JavaScript logic over a modern Svelte 5 reactive framework.

### Features
*   **Acoustic Realism**: Bypasses raw synthetic sine waves by integrating **Tone.js** with high-resolution acoustic samples from the Salamander Yamaha Grand Piano.
*   **Floating-Point Micro-Timing**: Interprets continuous, non-grid-locked human gesture from text inputs like `beat 1.63 duration 0.58`. Web Audio and Tone.js playback algorithms mathematically schedule audio graphs precisely without snapping/quantization.
*   **Dual-Channel Polyphony (Chords + Melody)**: Uses the **`tonal`** music theory library to intelligently resolve complex jazz chord nomenclatures (e.g. `Am7b5`, `D7alt`, `Bbmaj7#11`) into stacked chord sets played quietly beneath the leading melody line.
*   **Reactive Diagnostics**: Instantly updates UI metrics (BPM, Key, Notes Count, Chords Count) in real-time as the text prompt is edited.
*   **Binary MIDI Compilation (`.mid`)**: Click "MIDI" to instantly encode the entire text payload into a Format-1 Multi-Track MIDI blob directly in the browser. Tracks isolate Chords and Melody onto discrete channels to be dropped straight into any DAW.
*   **macOS Dark Mode**: Features a responsive Vanilla CSS desktop aesthetic mimicking macOS window borders, exclusively styled with proper structural viewport breakpoints.
*   **Svelte 5 "Runes" Architecture**: Refactored to use cutting-edge Svelte 5 `$state` and `$derived` compilation for maximal performance and zero legacy code warnings.

## 🚀 Getting Started

### Prerequisites
*   Node.js (v18+ recommended)
*   npm or pnpm

### Installation
1. Clone the repository and navigate into the folder:
   ```bash
   cd jazzPrompt2midi
   ```
2. Install dependencies (Vite, svelte, tonal, tone):
   ```bash
   npm install
   ```
3. Start the local development server:
   ```bash
   npm run dev
   ```

## 📝 Payload Formatting Reference

In order for the parser to correctly build the dual-track arrays, the input text *must* follow this explicit structural format. Note the blank lines and exact colon spacing. The default prompt is automatically loaded from `src/defaultPayload.txt`.

```text
TITLE: Your Song Name
TEMPO: 68 BPM
KEY: F major
TIME: 4/4
SWING: light
BARS: 16

CHORDS
bar 1: Bb6/9(4, velocity 88)
bar 2: G7(b9)(2) C7(2) velocity 82
bar 3: Fm7 (2) Bb7 (2)
bar 4: Ebmaj9 (4)
bar 8: Bb6(4, velocity 104)

MELODY
bar 1 beat 1.0: F4 duration 1.0 velocity 82
bar 1 beat 1.89: C5 duration 0.62 velocity 90
bar 2 beat 2.41: C5 duration 0.93
```

### Formatting Rules
1.  **Metadata**: Must declare `TEMPO: [number]`, `TIME: [num/den]`, and `BARS: [number]`.
2.  **`CHORDS` Section**: Must start with the literal word `CHORDS`.
    *   Lines format: `bar [number]: [Chord1] [Chord2]... [velocity X]`
    *   By default, chords on the same line divide the measure equally.
    *   To specify beat durations and velocity, trail them with parentheses: `Cm7(3)` or `Cm7(3, velocity 88)`.
    *   Chord names with internal parentheses (like `G7(b9)`) are fully supported.
3.  **`MELODY` Section**: Must start with the literal word `MELODY`.
    *   Lines format: `bar [number] beat [float-position]: [PitchWithOctave] duration [float] velocity [int]`.
    *   Velocity is optional (defaults to ~80).

## 🛠️ Tech Stack & Philosophy
*   **Framework**: Vite + **Svelte 5**
*   **Reactivity System**: Svelte Runes (`$state`, `$derived`, `onclick`)
*   **Styling**: Pure semantic HTML + Vanilla CSS `<style>` (No Tailwind)
*   **Audio Synthesis**: The `Tone.js` library utilizing the `Tone.Sampler` class routed precisely using `Tone.now()`.
*   **Harmony Interpolation**: `tonal` (tonaljs)
*   **Binary Serialization**: Vanilla JavaScript Bitwise operations array buffer `Uint8Array`. No bulky external MIDI NPM libraries.
